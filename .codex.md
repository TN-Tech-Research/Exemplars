## Poster Asset Pipeline

The app does not generate poster assets itself. It consumes prepared assets from:

- Source asset directory: `/home/bates/Pictures/RCIS/Exemplars/IMG`

The sync step:

- copies all `*.png` and `*.json` files from that source into `public/posters`
- generates `public/posters/catalog.json`

This is done by:

- `node scripts/sync-assets.mjs`

This sync is already wired into the build:

- `npm run build` runs `npm run sync:assets && tsc && vite build`

## Poster Data Shape

Each poster manifest in `public/posters/*.json` contains:

- `project_number`
- `project_title`
- `project_authors`
- `project_authors_display`
- `abstract`
- `ref`
- `pdf_filename`
- `image_filename`

The generated `catalog.json` adds:

- `image_path`
- `manifest_path`

Current catalog count:

- 34 posters

## Upstream Asset Prep Context

Poster assets were prepared outside this repo from the RCIS exemplar PDFs.

That prep produced:

- high-resolution PNG poster images
- sidecar JSON manifests

The source prep logic exists at:

- `/home/bates/Pictures/RCIS/Exemplars/generate_img_assets.py`

Important upstream join logic:

- `PDF/<Project Number>.pdf`
- joined to `FinalProjects.csv` on `Project Number`
- then joined to `Projects.csv` on `Ref -> ID`
- abstract comes from `Projects.csv`
- authors come from `FinalProjects.csv` and were normalized from `Last, First (id)` to `First Last`

Known upstream exception:

- `ASC-260` had a blank `Ref` in `FinalProjects.csv`
- a documented override was used to map it to `Projects.csv` ID `248`
- this has already been resolved in the prepared manifests
